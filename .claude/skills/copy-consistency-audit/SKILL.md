# Copy Consistency Audit — 繁中文案一致性審核

**觸發指令:** 審核中包含「文案」「用語」「一致性」「copy」關鍵字時自動載入

---

## 職責

審核所有使用者可見的繁體中文（台灣用語）文案，確保產品術語、語氣、格式在整個專案中一致。不審核代碼品質（那是其他 skill 的事），只審核「使用者看到的文字」。

---

## 產品術語對照表（Single Source of Truth）

以下術語在整個專案中必須統一，不得混用：

| 正確用語 | 禁止用語 | 說明 |
|---------|---------|------|
| 安心留痕 | 安全交易、信任服務、信任驗證 | 產品核心功能名稱 |
| 經紀人 | 房仲、業務、仲介 | 對外統一稱呼（代碼內部 `agent` 可保留） |
| 加 LINE 聊聊 | LINE 聯繫、LINE 諮詢、加入 LINE | CTA 按鈕文案 |
| 致電諮詢 | 撥打電話、打電話、電話聯繫 | CTA 按鈕文案 |
| 預約看屋 | 預約參觀、預約賞屋、預約看房 | CTA 按鈕文案 |
| 送出諮詢 | 送出、提交、確認送出 | 表單送出按鈕 |
| 邁房子 | MaiHouses、Mai Houses | 品牌名稱 |

## 語氣一致性規則

### 1. 敬語規則：統一使用「你」

**規則:** 全站對消費者使用「你」而非「您」。「您」只在極正式場合（法律條款、隱私政策）使用。

**掃描:** Grep `您` 在 `.tsx` 檔案中的出現，逐一判斷是否應改為「你」。

**例外:** `您好` 在 greeting 中可保留。

---

### 2. 時間承諾一致性

**規則:** 經紀人回覆時間承諾全站統一。不得出現「10 分鐘」和「30 秒」混用。

**掃描:** Grep `分鐘|小時|秒` 在使用者可見文案中的出現。列出所有時間承諾並比對。

---

### 3. 錯誤訊息風格

**規則:**
- 標題：簡短說明發生什麼（例：「送出失敗」）
- 描述：告訴使用者下一步（例：「請稍後再試」）
- 語氣：冷靜友善，不責怪使用者

**禁止:**
```
"系統錯誤"  → 太技術性
"操作失敗"  → 太冷漠
"Error"     → 英文不該出現在使用者面前
"請重試"    → 太命令式
```

**正確範例:**
```
notify.error('送出失敗', '請稍後再試');
notify.warning('操作未完成', '請稍後再試');
notify.success('已建立安心留痕', '交易紀錄已同步建立。');
```

---

### 4. Placeholder 格式

**規則:** 所有 input placeholder 使用「例如：XXX」或直接提示格式。

**統一格式:**
```
電話: "0912-345-678"
姓名: "如何稱呼你"
需求: "例如：想了解學區、需要停車位..."
```

---

### 5. Emoji 使用規範

**規則:** 使用者面向的 UI 禁止使用 emoji 作為 icon。

**允許:**
- 開發用 UI（`isDemo` 區塊內）
- 純文字內容中的表情（非替代 icon 用途）

**禁止:**
```tsx
// BAD: emoji 替代 icon
<p>📋 您的諮詢資訊已送達：</p>
<li>✅ 首次聯繫時間已記錄</li>

// GOOD: Lucide icon
<p><ClipboardList size={14} /> 你的諮詢資訊已送達：</p>
<li><CheckCircle size={12} /> 首次聯繫時間已記錄</li>
```

---

### 6. 按鈕文案長度

**規則:** 手機端按鈕文案 <= 4 個中文字（含 icon 時 <= 3 字）。

| 正確 | 太長 |
|------|------|
| LINE | 加 LINE 聊聊 (手機按鈕內) |
| 致電 | 致電諮詢 (手機按鈕內) |
| 預約 | 預約看屋 (手機按鈕內) |
| 預約看屋 | (桌面版可以用全稱) |

**注意:** 桌面版 CTA 可用全稱，手機版用簡稱。檢查 `lg:hidden` 區塊內的按鈕文案。

---

### 7. 數字格式

**規則:**
- 金額：「1,000 萬」（加千分位 + 空格 + 單位）
- 時間：「10 分鐘內」（數字 + 空格 + 單位 + 內）
- 人數：「32 組預約中」（數字 + 空格 + 量詞）
- 評分：「4.8」（一位小數）

---

## 評分規則

| 分類 | 權重 |
|------|------|
| 產品術語一致 | 30% |
| 語氣一致（敬語/時間/錯誤訊息） | 25% |
| Emoji 規範 | 15% |
| 格式一致（placeholder/數字/按鈕長度） | 15% |
| 無英文外洩到使用者面前 | 15% |

---

## 掃描流程

1. **Grep 術語表:** 掃描所有 `.tsx` 中的使用者可見字串，比對術語對照表
2. **Grep 敬語:** 搜尋 `您` 出現位置，判斷是否應改為「你」
3. **Grep 時間承諾:** 搜尋 `分鐘|小時|秒` 在 JSX 字串中的出現
4. **Grep Emoji:** 搜尋 Unicode emoji 範圍 `[\u{1F300}-\u{1F9FF}]` 和常見 emoji（📋✅❌🧪）
5. **逐檔檢查:** 每個問題標注 file:line + 原始文案 + 建議修正

