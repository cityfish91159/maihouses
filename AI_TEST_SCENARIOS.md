# 邁房子 AI 對話測試腳本

用於驗證 AI 引導是否自然、功能觸發是否準確。

---

## 測試目標

### ✅ 成功標準：
1. **語氣自然**：不讓用戶感到被推銷
2. **分層引導**：第 1-3 次提及社區，引導強度遞增
3. **功能準確**：正確觸發 F1-F8 功能
4. **邊界清晰**：允許/引導/限制範圍處理得當

### ❌ 失敗標準：
1. 第一次就強推社區牆（太急）
2. 回答過於公式化（像客服）
3. 誤觸發功能（例如聊天卻突然跳出貼文潤飾）
4. 對限制範圍直接說「我不會」（冷漠）

---

## 測試場景

### 場景 1：停車問題（分層引導測試）

**目標：** 驗證第 1-3 輪引導強度遞增

#### 第 1 輪
```
用戶輸入：這社區停車方便嗎？

✅ 期望回應：
- 先同理「停車確實很重要」
- 提供專業建議（車位配比、訪客車位等）
- 輕描淡寫提及「有住戶分享過」
- ❌ 不應立即觸發 scroll_to

評分標準：
[ ] 語氣溫暖自然
[ ] 有提供實用資訊
[ ] 沒有強推社區牆
```

#### 第 2 輪（用戶追問）
```
用戶輸入：訪客車位會很難停嗎？

✅ 期望回應：
- 具體化：「上個月有 3 位住戶討論過」
- 給出片段資訊（例如「假日較緊張」）
- 溫柔詢問：「要不要看看他們的經驗？」
- ❌ 不應強制觸發，而是引導

評分標準：
[ ] 提供更具體資訊
[ ] 引導語氣不強硬
[ ] 有提到具體數字/案例
```

#### 第 3 輪（用戶同意或繼續問）
```
用戶輸入：好啊，在哪裡看？
或：附近有沒有備用停車場？

✅ 期望回應：
- 主動觸發 scroll_to
- 或：「我幫您整理好了...」+ scroll_to

評分標準：
[ ] 正確觸發 scroll_to
[ ] JSON 格式正確
[ ] 文字說明流暢
```

### 場景 2：貼文潤飾（F1 功能測試）

**目標：** 驗證觸發準確度 + 內容品質

#### 測試 2-1：明確要求潤飾
```
用戶輸入：
幫我潤飾這段：「管委會都不管事，垃圾堆滿大廳，住戶素質很差」

✅ 期望回應：
- 觸發 community_post_refine
- 兩版本：normal（誠懇） + calm（冷靜）
- 避免毀謗：不直接說「某某人」
- 保留核心訴求：環境維護問題

評分標準：
[ ] 正確觸發 F1
[ ] 兩版本風格明顯差異
[ ] 用語避免攻擊性
[ ] JSON 格式正確
```

#### 測試 2-2：聊天轉貼文（F2）
```
（前面已對話 5 輪，討論某社區的優缺點）

用戶輸入：
把我們剛剛聊的整理成社區評價

✅ 期望回應：
- 觸發 community_post_refine
- 從歷史對話提取：社區名、優缺點、具體例子
- 組成完整貼文結構

評分標準：
[ ] 正確觸發 F2
[ ] 提取歷史資訊準確
[ ] 貼文邏輯完整
[ ] 標題和內文匹配
```

### 場景 4：價格詢問（溫柔引導測試）

**目標：** 驗證如何不直接回答，但也不冷漠拒絕

```
用戶輸入：這間房子 1800 萬會不會太貴？

✅ 期望回應：
- 不直接評論價格
- 轉向：「建議先看社區評價」
- 提供價值：列出隱藏成本（管理費、公設等）
- 引導：「要不要了解一下實際居住體驗？」

❌ 失敗案例：
- 「我不能評論價格」（太冷漠）
- 「這個價格很合理」（越界回答）
- 直接跳轉到社區牆（太硬）

評分標準：
[ ] 語氣溫暖
[ ] 成功轉向而不生硬
[ ] 有提供替代價值
[ ] 沒有直接評論價格
```

### 場景 5：法律問題（明確限制測試）

**目標：** 驗證如何禮貌拒絕 + 提供替代方案

```
用戶輸入：鄰居一直製造噪音，我可以告他嗎？

✅ 期望回應：
- 先同理「能理解您的困擾」
- 明確拒絕：「法律問題建議諮詢律師」
- 提供替代：「我能幫忙的是分享其他住戶的處理經驗」
- 列出 2-3 個實用方向（管委會、溝通技巧）

❌ 失敗案例：
- 「抱歉我不會」（太冷）
- 給出法律建議（越界）
- 完全不轉向（卡住對話）

評分標準：
[ ] 有同理心
[ ] 明確但不冷漠地拒絕
[ ] 提供替代方案
[ ] 對話能繼續進行
```

### 場景 6：看房邀請文案（F8 功能測試）

```
用戶輸入：幫我寫一段介紹這間房給老婆看

✅ 期望回應：
- 觸發 invite_text
- 文案長度 60-120 字
- 語氣溫和（給家人用）
- 包含：格局/坪數/社區名 + 1-2 亮點

評分標準：
[ ] 正確觸發 F8
[ ] 文案長度符合
[ ] 語氣適合對象（家人 vs 朋友）
[ ] 不推銷，只描述
```

### 場景 7：社區牆導航（F6 功能測試）

#### 測試 7-1：直接詢問
```
用戶輸入：社區牆在哪裡？

✅ 期望回應：
- 解釋：「社區牆是真實住戶分享的地方...」
- 觸發 scroll_to
- target: "community-wall"

評分標準：
[ ] 正確觸發 F6
[ ] 有解釋功能
[ ] 滾動行為正確
```

#### 測試 7-2：第 3 次提到同一社區（主動觸發）
```
第 1 輪：這社區停車方便嗎？
第 2 輪：管理費多少？
第 3 輪：附近有公園嗎？

✅ 期望回應：
- 在第 3 輪主動建議：「這社區您很感興趣，社區牆有完整資訊...」
- 觸發 scroll_to

評分標準：
[ ] 能追蹤社區提及次數
[ ] 第 3 次主動觸發
[ ] 引導自然不突兀
```

## 壓力測試

### 測試 8：混亂輸入

```
用戶輸入：
我想找台北的房子，預算 2000 萬，要有停車位，小孩要上學，
老婆說要近捷運，還要管理好，對了幫我潤飾一段社區評價

✅ 期望回應：
- 先理清需求：「您的需求我記下來了...」
- 拆解重點：區域/預算/停車/學區/捷運/管理
- 詢問：「我們可以先從區域開始，還是要先處理貼文潤飾？」

評分標準：
[ ] 沒有被混亂輸入搞懵
[ ] 能拆解多個需求
[ ] 引導用戶分步處理
```

---

### 測試 9：情緒化輸入

```
用戶輸入：
這個鄰居真的超討厭，每天吵到半夜，管委會都不處理，
我快崩潰了！你說我該怎麼辦！！！

✅ 期望回應：
- 先安撫：「真的很辛苦，這種情況確實會影響生活品質」
- 不直接給法律建議
- 提供實用方案：「不少住戶遇過類似問題，他們是這樣處理的...」
- 詢問：「要不要看看其他人分享的經驗？」

評分標準：
[ ] 有同理心
[ ] 語氣冷靜但溫暖
[ ] 沒有被情緒帶走
[ ] 提供建設性方案
```

## 評分表（綜合評估）

### A 級（優秀）：90-100 分
- 語氣自然，像真人對話
- 引導不著痕跡
- 功能觸發準確無誤
- 邊界處理溫暖有智慧

### B 級（良好）：75-89 分
- 功能基本準確
- 語氣偶有機械感
- 引導稍顯生硬但可接受

### C 級（及格）：60-74 分
- 功能有誤觸發或漏觸發
- 語氣明顯像客服
- 引導太強或太弱

### D 級（不及格）：< 60 分
- 功能觸發錯誤多
- 語氣冷漠或推銷感重
- 邊界處理不當
